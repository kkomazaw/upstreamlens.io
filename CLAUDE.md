# Upstream Lens  
## GitHub Pages サイト構築 要件定義書

---

## 1. 背景・目的

### 1.1 背景

CNCF を中心としたクラウドネイティブおよび Platform Engineering 領域では、  
SIG / WG / Project / Tool といった多層的な意思決定が日々行われている。

しかし多くの情報発信は以下に偏りがちである。

- 個別ツールやプロダクトの解説
- 実装・運用ノウハウ
- ベストプラクティスの断片的共有

その結果、  
**「なぜその判断がなされたのか」という upstream の構造や文脈が見えにくい**。

---

### 1.2 目的

Upstream Lens は以下を目的とする。

- CNCF を含む upstream コミュニティにおける意思決定を  
  **構造・文脈・視点**から読み解く
- 技術トレンドを「結果」ではなく  
  **意思決定プロセスとして理解できる場を提供する**
- 筆者自身の思考軸を明示した上で、  
  **キュレーションと解釈を分離した情報発信を行う**

---

## 2. サイトの基本コンセプト

### 2.1 サイト名

- **Upstream Lens**

### 2.2 タグライン

- *Seeing decisions before code is written*

---

### 2.3 編集方針（Editorial Principle）

- 中立を装わない
- 視点を先に固定する
- 事実（Facts）と解釈（Interpretation）を明確に分離する
- upstream → downstream の流れを重視する

---

## 3. 思考フレームワーク（中核要件）

### 3.1 基本構造（最重要要件）

本サイトは、以下の **3軸構造**を常に前提として扱う。

| 軸 | 説明 |
|---|---|
| Product | 価値仮説、ユーザー、提供したい体験 |
| Engineering | 技術選択、設計判断、実装可能性 |
| Organization / Architecture | 組織構造、運用、ガバナンス、持続性 |

すべての記事・分析は、

> この3軸のどこで、どのようなトレードオフが発生しているか

を暗黙または明示的に扱う。

---

### 3.2 CNCF との関係付け

- SIG / WG / Project は  
  **技術分類ではなく「意思決定の成熟度」として捉える**
- Project は完成形ではなく、  
  **複数の制約と妥協の結果として成立しているもの**として分析する

---

## 4. 対象読者

### 4.1 主対象読者

- Platform Engineer
- Enterprise / Cloud Architect
- OSS コミュニティの構造に関心を持つ読者
- 技術選定や標準化に関与するリード・マネージャ層

---

### 4.2 対象外とする読者

- 初学者向けハンズオンやチュートリアルを求める読者
- 単一ツールの使い方解説を主目的とする読者
- SEO や速報性のみを期待する読者

---

## 5. コンテンツ要件

### 5.1 コンテンツ種別

| 種別 | 内容 |
|---|---|
| Philosophy | サイトの思想・視点の明文化 |
| Notes | upstream 観察メモ |
| Analysis | 意思決定構造の分析 |
| Curation | 外部情報の整理と解釈 |
| Meta | コミュニティ構造や進化の考察 |

---

### 5.2 事実と解釈の分離（必須要件）

各記事では以下を区別する。

- **Facts**
  - CNCF 公式文書
  - コミュニティ議論
  - Issue / PR / 発言など一次情報
- **Interpretation**
  - 筆者の視点による構造化・意味付け

---

## 6. サイト構成要件

### 6.1 必須ページ

| ページ | 目的 |
|---|---|
| Home | サイトコンセプトの提示 |
| About | サイトの立ち位置説明 |
| Philosophy | 思考軸の明示 |
| How to Read | 読み方・前提の共有 |
| Disclaimer | 免責事項 |
| Author | 筆者紹介 |

---

### 6.2 トップページ要件

- 強いコピーを配置する
- 技術用語の羅列を避ける
- 「何を読むサイトか」が即座に理解できること

---

## 7. 技術要件

### 7.1 ホスティング

- GitHub Pages
- 独自ドメイン：`upstreamlens.io`

---

### 7.2 技術スタック

| 項目 | 採用 |
|---|---|
| Static Site Generator | Jekyll |
| Theme | GitHub Pages 標準テーマ |
| CMS | 使用しない |
| コメント機能 | 当面は提供しない |

---

### 7.3 非機能要件

- 高速に表示されること
- 長期的に保守可能であること
- 外部依存を最小限に抑えること
- Markdown 中心で執筆できること

---

## 8. 法務・ポリシー要件

### 8.1 OSS・ロゴ利用

- CNCF および各プロジェクトのロゴ・商標ポリシーを遵守する
- 本サイトが公式ではないことを明記する

---

### 8.2 免責事項

- 本サイトの内容は筆者個人の見解である
- 正確性・完全性を保証しない
- 投資・導入判断を目的とした情報提供ではない

---

## 9. 成功の定義

数値的 KPI ではなく、以下を重視する。

- 読者が upstream の視点を持ち帰れること
- 他の技術記事や議論を異なる角度で読めるようになること
- 意思決定構造を言語化できる人が増えること

---

## 10. 将来拡張（スコープ外）

- 英語版サイトの提供
- CNCF イベント・リリースに合わせた特集
- SIG / WG / Project の関係性可視化
- 長編エッセイ・レポート化

---

## 11. 非目標（やらないこと）

- 広告による収益化
- バズを狙った記事作成
- ツール比較ランキング
- ニュース速報型メディア化

---

## 結語

Upstream Lens は、  
正解を提示するサイトではなく、  
**思考の前提を固定するためのレンズ**である。
